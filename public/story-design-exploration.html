01 · Research

What the best narrative experiences teach us

Synthesized from scrollytelling UX research, educational media design, and premium interactive journalism patterns.

📍

Progressive disclosure, not pagination

Learners should feel they're uncovering a story, not clicking through slides. Scroll-coupled visuals create "revelation moments" where the diagram transforms as understanding deepens.

Scrollytelling · NYT, Pudding
🧭

Wayfinding reduces cognitive load

Named beats ("GPS for any space" → "Dimension") beat anonymous dots. Learners always know where they are in the journey and what's coming next.

NN/g · Progress indicators
🎭

Character as guide, not decoration

Vera should feel present — voice bubbles, reactions to predictions, celebration on completion. Characters create emotional anchors that improve retention.

Educational narrative design
⚡

Motion with purpose

Every animation should communicate state change: beat advance, passage activation, diagram interpolation. Avoid decorative motion; use 400–680ms expo easing for "premium" feel.

Material · Apple HIG
📐

Typographic hierarchy is the UI

In reading-heavy experiences, type scale does more work than chrome. Distinct serif for prose, mono for wayfinding, generous line-height (1.65–1.75) for comprehension.

Butterick's Practical Typography
🎯

Commitment before content

Predict-before-read creates active engagement. The UI should make this feel like a meaningful wager, not a quiz — with character nudges that reference the learner's choice.

Desirable difficulties · Bjork

11 · System

Typography & motion tokens

Proposed type scale and easing curves for consistent implementation across all 10 characters.

Design tokens
Display
48–72px Basis & Dimension
Heading
32–40px A GPS for any space
Body
19–22px A basis is like having the perfect GPS coordinate system — the minimum set of independent vectors that can reach everywhere.
Caption
11px mono Vera · Vectors · Beat 1

Motion timing

Fast: 180ms — hovers, toggles
Medium: 420ms — beat transitions, passage activation
Slow: 680ms — cold open, completion
Easing: cubic-bezier(0.16, 1, 0.3, 1)

Character theming

Each character gets accent pair + glow intensity. Stage background stays constant; only accent radiants shift. Vera = red/orange; Max = blue/cyan; etc.

Implementation path

1. Typography + tokens in CSS
2. Cold open + wayfinding components
3. CoupledVisual polish
4. Character voice layer
5. Mobile diagram UX
6. Per-character QA pass
